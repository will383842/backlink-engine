## 8. UTILISATION DES 2 IPs

### Faut-il PMTA sur .60 ? **NON** court/moyen terme.
1. Licence crackée à dupliquer = risque légal doublé
2. MailWizz sait parler SMTP submission natif, n'a pas besoin de PMTA
3. Licence officielle PMTA ~$2k/an × 2 = non rentable pour cold outreach

### Architecture recommandée — **Option A : Postfix sur .60 comme 2e IP outreach**

```
[BL Engine]                  [MailWizz .60]
     │                             │
     ▼                             ▼
[Postfix:10025] .175      [Postfix submission] .60
     │ OpenDKIM                    │ OpenDKIM
     ▼                             ▼
[PMTA:2525] .175          [Postfix smtp] .60 → direct → ISPs
     │
     ▼
[ISPs via 204.168.180.175]
```

**Avantages** :
- Split réputation : si .175 re-liste, .60 continue
- Pas de licence PMTA supplémentaire
- MailWizz peut enfin envoyer via delivery_server local
- Isolation campagnes : BL Engine sur .175, MailWizz sur .60

**Prérequis** :
- Installer Postfix + OpenDKIM sur .60 (10 min)
- Créer PTR `mail2.life-expat.com` pour .60 (Hetzner panel, 5 min)
- Cert Let's Encrypt pour `mail2.life-expat.com` (5 min)
- SPF des 5 domaines : `v=spf1 ip4:204.168.180.175 ip4:204.168.254.60 -all` (⚠️ APRÈS DELISTING)
- Dupliquer DKIM keys ou créer selector distinct (`dkim2`)
- MailWizz delivery_server type `smtp`, `127.0.0.1:587`

**Option B — Tunnel SSH MailWizz(.60) → PMTA(.175)** : défaut = single point of failure, .60 inutile en split.

**Option C — Déplacer MailWizz sur .175** : économise VPS2 mais perd split IP.

### 9.2 NOUVEAU : `/opt/mail-security/pre-send-check.sh`

```bash
#!/bin/bash
# Pre-send safety check
# Exit 0 = GO, Exit 1 = STOP
set -e
REASONS=""

SPAM=$(cat /var/lib/mail-security/spamhaus-state 2>/dev/null || echo unknown)
[ "$SPAM" != "clean" ] && REASONS="$REASONS spamhaus=$SPAM;"

MS=$(cat /var/lib/mail-security/microsoft-state 2>/dev/null || echo unknown)
[ "$MS" != "clean" ] && REASONS="$REASONS microsoft=$MS;"

ufw status 2>/dev/null | grep -q "Status: active" || REASONS="$REASONS ufw=inactive;"

for svc in pmta postfix dovecot opendkim; do
  systemctl is-active --quiet "$svc" || REASONS="$REASONS $svc=down;"
done

USED=$(df / --output=pcent | tail -1 | tr -dc '0-9')
[ "$USED" -ge 85 ] && REASONS="$REASONS disk=${USED}%;"

ACCT="/var/log/pmta/acct-$(date -u +%Y-%m-%d)-0000.csv"
if [ -f "$ACCT" ]; then
  SINCE=$(date -u -d '1 hour ago' +"%Y-%m-%d %H:%M")
  D=$(awk -F, -v t="$SINCE" 'NR>1 && $2>t && $1=="d"' "$ACCT" | wc -l)
  B=$(awk -F, -v t="$SINCE" 'NR>1 && $2>t && $1=="b"' "$ACCT" | wc -l)
  TOTAL=$((D+B))
  if [ "$TOTAL" -gt 20 ]; then
    RATE=$((B * 100 / TOTAL))
    [ "$RATE" -ge 5 ] && REASONS="$REASONS bounce_1h=${RATE}%;"
  fi
fi

ss -tlnp 2>/dev/null | grep -qE "204\.168\.180\.175:2525|0\.0\.0\.0:2525" && REASONS="$REASONS pmta_public;"

if [ -n "$REASONS" ]; then
  echo "STOP: $REASONS"
  exit 1
fi
echo "GO"
exit 0
```

```bash
  NR>1 && $2>t {
    v=$19
    if ($1=="d") delivered[v]++
    else if ($1=="b") bounced[v]++
  }
  END {
    for (v in delivered) {
      total=delivered[v]+bounced[v]
      if (total>=10 && v!="vmta-hub") {

        if (rate>=5) printf "%s: %.1f%% (%d/%d)\n", v, rate, bounced[v], total
      }
    }
    for (v in bounced) {
      if (!(v in delivered) && bounced[v]>=5 && v!="vmta-hub") {
        printf "%s: 100%% (%d bounces)\n", v, bounced[v]
      }
    }
  }
' "$ACCT" > /tmp/vmta-bounce-tmp

if [ -s /tmp/vmta-bounce-tmp ]; then

  NEW=$(cat /tmp/vmta-bounce-tmp)

    MSG="⚠️ VMTA BOUNCE ALERT%0A%0A$(cat /tmp/vmta-bounce-tmp | tr '\n' '|')"

    echo "$NEW" > $STATE
  fi
fi
```
